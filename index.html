Core principle: correct nothing except subject-verb agreement violations. This prevents over-correction, hallucination, and unintended meaning changes.

TWO-STAGE PIPELINE
01
Input Preprocessing
Tokenization → POS tagging → Dependency parsing → Constituency parsing. The sentence is structurally decomposed to identify all subject-verb pairs and their grammatical number.
02
Stage 1 — SVA Detection Module
A meta-learning ensemble of 7 neural models (LSTM, GRU, BiLSTM, Conv1D, DistilBERT, RoBERTa, XLNet) classifies each verb token as SVA-correct or SVA-incorrect. Blending strategy achieves 92.88% accuracy.
03
Precision Gate — Key Innovation
Only sentences where SVA errors are confidently detected proceed to Stage 2. Correct sentences are returned unchanged — eliminating false corrections and over-generation.
04
Stage 2 — Targeted Correction
A fine-tuned T5 sequence-to-sequence model corrects only the flagged SVA errors using grammar prefix formatting. Fine-tuned on Lang-8 learner corpus (5 epochs, lr=3×10²⁻⁴, FP16).
05
Output Verification
Beam search decoding (beams=4, max_len=128). Post-processing preserves original capitalisation and punctuation. Only SVA-related tokens are modified.

RULE 2
Plural subject → plural verb

RULE 3
Compound subjects (and) → plural
Ram and Shyam is → are
RULE 4
Or/Nor → agree with nearest subject
Neither he nor they was → were
RULE 5
Each / Every → singular verb
Each students are → is
RULE 6
Collective nouns → singular
The team are ready → is
RULE 7
Indefinite pronouns → singular

RULE 8
There is/are mismatch

RULE 9
Relative pronoun clause SVA
The man who run → runs
RULE 10
Prepositional phrase interruption
A box of chocolates are → is
RULE 11
Gerund / infinitive as subject → singular
Running are fun → is
RULE 12
Amounts / Titles → singular
Five dollars are too much → is
WHY ONLY SVA — NOT ALL ERRORS

Traditional GEC systems correct all errors indiscriminately, causing: (1) Over-correction of stylistic choices, (2) Hallucination — unintended meaning changes, (3) False positives in domain-specific text. The two-stage Detect-then-Correct design activates the generative model only when an SVA error is confidently identified — achieving high-precision correction with zero unintended side effects.
